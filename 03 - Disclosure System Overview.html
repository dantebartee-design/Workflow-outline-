File Naming, Folders & Revision Tracking

Print this and keep it at your desk. One page, everything you need to file a disclosure correctly.

1

Ground rules — file naming

20260601 initial disclosure statement (D L)
Date — YYYYMMDD (1 June 2026) Description — what the document is Initials — who has touched it
  • Date first, always. YYYYMMDD keeps files in chronological order automatically.
  • Plain description in lowercase — say what it is, no codes.
  • Initials in parentheses track revisions: (D) = Dante drafted, (D L) = Dante drafted, Lawyer reviewed.
  • Add initials, don't replace. The trail grows left-to-right as people touch it.
  • One document, one name — update the same file; don't spawn "final_final_v3".
  • No special characters beyond spaces and the parentheses.

Folder structure & how documents flow

Revision tracking — how the trail builds

Example: one document's life, start to finish

20260601 initial disclosure statement (D)Dante — first draft“First pass from template — flagged 2 gaps for Holly.”
20260601 initial disclosure statement (D L)Holly — reviewed“Tightened para 3; confirm exhibit B date.”
20260601 initial disclosure statement (D L D)Dante — revised“Para 3 updated; exhibit B date confirmed 28/05.”
20260601 initial disclosure statement (D L D L)Holly — approved ✓“Approved. Move to Done & export to Clio.”
How notes get added: keep a short note with each change — a comment in the doc, or a one-line entry in a running log. The initials in the filename tell you who last touched it; the note tells you what changed. Together they are the revision history, no version-number guessing required.
